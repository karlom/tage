Product Requirements Document: Tage AI (Business Pilot Edition)Version: 2.0 (Commercial Pivot)Date: 2025-12-12Target Audience: 中小企业主、高管、超级个体 Core Philosophy: 零配置、场景化、隐私优先 1. 项目概述 (Project Overview)产品定义：Tage 是一款面向非技术背景企业主的 "桌面端数字幕僚"。它隐藏了所有底层 AI 技术细节，通过预设的数字员工（Digital Staff）和私有知识库（Private Vault），协助老板完成决策辅助、商务沟通和信息处理。商业核心价值：开箱即用 (Zero-Config): 去除 API Key 申请、模型参数调整等门槛，下载即用（通过内置代理服务或极简订阅）。数据主权 (Local First): 强调“商业机密不出本地”，所有 RAG 索引和文档存储均在本地完成。结果导向 (Result-Oriented): 不提供“聊天”，只提供“解决方案”。2. 核心体验重构 (Core Experience Shift)维度原技术 PRD (V1.0)现商业 PRD (V2.0)用户心智管理 AI 模型的客户端随时待命的数字董事会交互入口新建聊天 (New Chat)呼叫员工 (Call Staff)参数设置 Temperature, Top P, Token 严谨 / 创意 / 激进 (模式选择)知识库向量检索配置, 相似度阈值投喂资料 (Feed Data)快捷键触发功能触发业务流 (一键回怼、一键总结)3. 功能模块详解 (Functional Modules)3.1 首页：数字董事会 (Digital Boardroom)(替代原有的 Sidebar + 空白聊天页)布局逻辑： 类似于即时通讯软件的联系人列表，而不是枯燥的文件树。预设角色 (Pre-loaded Personas):首席法务 (Legal): 预设 System Prompt 侧重风险审查、合同漏洞挖掘。金牌销售 (Sales): 侧重高情商话术、逼单技巧、客户安抚。战略参谋 (Strategy): 侧重宏观分析、SWOT 分析（也是东波哥的数字分身）。通用助理 (Assistant): 处理杂事、翻译、总结。交互： 点击头像直接进入对话，界面顶部显示该角色的“人设标签”（如：“当前为法务模式，我会严格审查每一句话”）。3.2 投喂中心 (The Vault)(替代原有的 Memory Settings)视觉隐喻： 一个保险箱或文件夹图标，暗示“安全”。拖拽投喂： 支持直接将 PDF、Word、PPT、Excel 拖入。自动分类： 系统自动识别文件类型并打标（例如：#公司介绍 #产品报价 #历史合同）。隐私承诺： 界面显著位置常驻提示：“文件仅建立本地索引，并未上传云端，甚至连 Tage 官方也无法查看。”3.3 全局“一键”悬浮球 (Magic Overlay)(替代原有的 Shortcuts)触发方式： Cmd + Space (或双击 Ctrl) 唤起全局搜索框/操作栏。场景化指令 (Contextual Actions):"帮我回这段话": 选中微信/钉钉里的一段文字 -> 唤起 Tage -> 选择态度（强硬、委婉、拒绝） -> 生成回复。"这是啥意思": 选中一段专业术语或英文 -> 唤起 Tage -> 解释人话。"整理会议": 粘贴乱七八糟的语音转文字 -> 生成 Action Items。老板专属“吐槽生成器” (病毒传播点): 输入“员工迟到还顶嘴”，生成一段既有威严又不失幽默的敲打文案，并支持一键生成精美卡片分享到朋友圈。3.4 极简设置 (Settings Lite)账户管理： 手机号一键登录（关联微信支付，用于订阅高级会员）。模型切换（隐藏式）： 默认使用最优模型（如 Claude 3.5 Sonnet 或 GPT-4o），用户无需选择。注：保留“极客模式”入口（点击版本号 5 次开启），供技术流老板自定义 API Key 和 Base URL。声音与皮肤：保留机械键盘音效（解压）。增加“商务深色”与“禅意白”主题。4. 技术栈调整 (Tech Stack Refinement)保持原有的 Electron + React + TypeScript 架构，但在数据层做以下优化：Default Providers: 后端需搭建一层 Proxy 服务（基于 Golang 或 Node.js），内置你的企业级 API Key 池。用户付费后，通过你的 Proxy 转发请求，从而实现“免配置”。Prompt Management: 所有的 System Prompt 不再硬编码，而是通过云端下发（JSON Config），方便你随时更新“金牌销售”的话术逻辑，实现 SaaS 化的运营。Local RAG: 继续使用 SQLite + 本地 Embedding 模型（推荐使用量化后的 bge-m3 或类似轻量模型），确保在没有 GPU 的商务本上也能跑得动。5. 数据结构更新 (Data Schema Update)TypeScript// 角色定义 (Personas)
interface Persona {
id: string; // e.g., 'legal_expert'
name: string; // "首席法务"
avatar: string; // 本地路径
description: string;
systemPrompt: string; // 核心资产，云端同步
style: {
tone: "formal" | "casual" | "aggressive"; // 对应原 Temperature
responseLength: "short" | "detailed";
};
attachedKnowledgeTags: string[]; // 自动挂载的知识库标签，如 ['contracts']
}

// 知识库文件 (Vault Items)
interface VaultItem {
id: string;
filename: string;
fileType: string;
category: "product" | "hr" | "finance" | "general";
vectorStatus: "indexed" | "pending" | "failed";
addedAt: number;
}

// 用户订阅 (User Subscription)
interface UserProfile {
id: string;
tier: "free" | "pro" | "enterprise"; // free 用户需填 Key，pro 用户免配置
tokensRemaining: number;
}
